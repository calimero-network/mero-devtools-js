ABI Format

WASM ABI v1 manifest — the JSON schema the codegen tool reads

Manifest Structure

A WASM ABI manifest is a JSON file that describes the public interface of a Calimero WASM contract: its methods, events, and named types.

interface AbiManifest {
  schema_version: 'wasm-abi/1';       // must be exactly 'wasm-abi/1'
  types: Record<string, AbiTypeDef>;  // named type definitions
  methods: AbiMethod[];            // callable contract methods
  events: AbiEvent[];             // emitted events
  state_root?: string;             // optional state root type reference
}

Minimal Example

{
  "schema_version": "wasm-abi/1",
  "types": {
    "SetEntryRequest": {
      "kind": "record",
      "fields": {
        "key": { "kind": "string" },
        "value": { "kind": "string" }
      }
    }
  },
  "methods": [
    {
      "name": "get_value",
      "params": [{ "name": "key", "type": { "kind": "string" } }],
      "returns": { "kind": "string" }
    },
    {
      "name": "set_value",
      "params": [{ "name": "req", "type": { "$ref": "SetEntryRequest" } }],
      "returns": { "kind": "unit" }
    }
  ],
  "events": []
}

Scalar Types

Primitive types are expressed as an object with a kind field:

{ "kind": "bool" }
Boolean
{ "kind": "string" }
UTF-8 string
{ "kind": "unit" }
Void / no return value
{ "kind": "i32" }
Signed 32-bit integer
{ "kind": "i64" }
Signed 64-bit integer
{ "kind": "u32" }
Unsigned 32-bit integer
{ "kind": "u64" }
Unsigned 64-bit integer
{ "kind": "f32" }
32-bit float
{ "kind": "f64" }
64-bit float
{ "kind": "bytes", "encoding": "hex" }
Byte array (hex-encoded). Add "size": N for fixed-length.

Collection Types

list
{ "kind": "list", "items": <type> } — array of items. Optional "crdt_type" for CRDT-backed lists.
map
{ "kind": "map", "key": <type>, "value": <type> } — key-value map.
tuple
{ "kind": "tuple", "elements": [<type>, ...] } — fixed-length heterogeneous tuple.
$ref
{ "$ref": "TypeName" } — reference to a named type in types.

Named Type Definitions

The types map defines reusable named types:

Record (struct)

{
  "kind": "record",
  "fields": {
    "name": { "kind": "string" },
    "count": { "kind": "u32" }
  }
}

Variant (enum)

{
  "kind": "variant",
  "cases": {
    "Pending": null,
    "Done": { "kind": "string" }
  }
}

Methods

{
  "name": "create_post",                        // snake_case method name
  "params": [
    { "name": "title", "type": { "kind": "string" } }
  ],
  "returns": { "$ref": "Post" }
}

Generated TypeScript method: createPost(title: string): Promise<Post>

Events

{
  "name": "PostCreated",
  "fields": {
    "id": { "kind": "string" },
    "title": { "kind": "string" }
  }
}

Events are emitted by the contract during execution and received via the SSE subscription. The codegen generates a typed payload interface: PostCreatedEvent.
