Architecture

Source structure and codegen pipeline internals

Codegen Pipeline

The tool reads a WASM ABI JSON file (exported by the Calimero Rust SDK) and emits TypeScript client code with typed method signatures and serialization helpers:

Source Structure

src/
All TypeScript source files for the CLI and codegen library.
src/cli.ts
Entry point — parses --input, --output, --name flags and orchestrates the pipeline.
src/parser.ts
Reads and validates the ABI JSON structure. Produces a typed in-memory representation of methods, arguments, and return types.
src/generator.ts
Maps ABI types to TypeScript types and renders the client class template with method stubs.
src/types.ts
Shared TypeScript interfaces for the ABI schema (AbiMethod, AbiParam, AbiType).
src/templates/
Handlebars or string-template files for the generated client wrapper and type helpers.

ABI Type Mapping

Rust types from the ABI JSON are mapped to TypeScript equivalents:

Primitive Types

u8 / u16 / u32 / u64
number / bigint
i8 / i16 / i32 / i64
number / bigint
bool
boolean
String
string

Complex Types

Vec<T>
T[]
Option<T>
T | null
HashMap<K,V>
Record<K, V>
struct / enum
Generated interface / type

Generated Output

Example: a Rust method fn get_post(id: u32) -> Post becomes:

// Generated — do not edit
export interface Post {
  id: number;
  title: string;
  body: string;
}

export class AppClient {
  constructor(private rpc: RpcClient) {}

  async getPost(id: number): Promise<Post> {
    return this.rpc.query({
      method: 'get_post',
      argsJson: JSON.stringify({ id }),
    });
  }
}
