Codegen Pipeline

1

Parse & Validate

parse.ts reads the ABI JSON and validates it against the embedded JSON Schema using AJV. Checks for dangling $ref references. Returns a typed AbiManifest object.

2

Model

model.ts defines the TypeScript interfaces that mirror the ABI schema 1:1. AbiManifest, AbiMethod, AbiEvent, AbiTypeRef, and all type definition variants.

3

Generate

generate/client.ts traverses the manifest and emits TypeScript source. Handles all type definition kinds (record, variant, alias, scalar, collection), byte encoding, and client method signatures.

Generated File Structure

The generator emits a single <ClientName>.ts file with four sections:

1. Imports
Imports RpcClient (or useExecute) from the configured import path.
2. Type definitions
One interface per record, one union type per variant, one type alias per alias definition. All referenced types.
3. CalimeroBytes
A minimal utility class for hex ↔ Uint8Array conversion, emitted if the manifest uses bytes types.
4. Client class
One typed method per ABI method. Each wraps rpc.execute() with correct serialisation.
// Example generated output for a method named 'get_value':
async getValue(key: string): Promise<string> {
  return await this.rpc.execute<string>({
    contextId: this.contextId,
    method: 'get_value',
    argsJson: { key },
  });
}
