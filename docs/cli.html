CLI Reference

Command-line options for generating TypeScript clients from WASM ABI JSON

Installation

Global install

npm install -g @calimero-network/mero-devtools-js
# or
pnpm add -g @calimero-network/mero-devtools-js

Run without installing

npx @calimero-network/mero-devtools-js generate \
  --input abi.json \
  --output src/client.ts

Command: generate

Generates a TypeScript client from a WASM ABI JSON file:

mero-devtools generate [options]
--input <path>
Required. Path to the ABI JSON file exported from a Calimero WASM application.
--output <path>
Required. Destination path for the generated TypeScript file.
--name <ClassName>
Name for the generated client class. Default: AppClient.
--rpc-import <path>
Import path for RpcClient. Default: @calimero-network/mero-js.
--no-types
Skip generating TypeScript interfaces for structs and enums. Useful when types are already defined elsewhere.
--watch
Watch --input for changes and re-generate automatically.

Full Example

# Export ABI from your Rust WASM app:
cargo build --target wasm32-unknown-unknown --release
calimero-sdk export-abi target/wasm32-unknown-unknown/release/app.wasm -o abi.json

# Generate TypeScript client:
npx mero-devtools generate \
  --input abi.json \
  --output src/generated/client.ts \
  --name TodoClient

# Use the generated client:
import { TodoClient } from './generated/client';
import { MeroJs } from '@calimero-network/mero-js';

const sdk = new MeroJs({ baseUrl: 'http://localhost:2428' });
const client = new TodoClient(sdk.rpc);
const todos = await client.getTodos();

Integrating with package.json

Add a codegen script to auto-generate on build:

// package.json
{
  "scripts": {
    "codegen": "mero-devtools generate --input abi.json --output src/client.ts",
    "build": "npm run codegen && vite build",
    "dev": "npm run codegen -- --watch & vite"
  }
}
